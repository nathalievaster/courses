Min arbetsprocess för utveckling och publicering

Den här sidan beskriver i detalj hur jag har arbetat med utveckling, automatisering och publicering av min webbplats. Jag har använt moderna verktyg och metoder för att effektivisera arbetsflödet och säkerställa en smidig process. Nedan går jag igenom de viktigaste stegen och verktygen jag har använt.

  1. Projektstart och strukturering

    Jag började projektet genom att skapa en grundläggande mappstruktur. Jag skapade en "src"-mapp där jag lade alla mina arbetsfiler (HTML, CSS, JavaScript och bilder). För att hålla projektet organiserat delade jag upp dessa filer i undermappar för att enkelt kunna navigera mellan dem.

  2. Installera Parcel

    För att börja med utvecklingen installerade jag Parcel, ett byggverktyg och bundlare för webben. Detta valde jag eftersom det automatiskt hanterar många saker som optimering av filer och byggprocessen utan att behöva konfigurera komplicerade inställningar. Jag installerade Parcel genom att köra npm install parcel i terminalen i mitt projekt.

  3. Skapa och strukturera HTML, CSS och JavaScript

    Jag skapade flera HTML-sidor som innehåller grundläggande strukturer för webbplatsen. För att stajla sidorna använde jag CSS, där jag definierade utseendet på elementen och la till responsiv design så att webbplatsen skulle fungera bra på olika enheter. För interaktivitet skapade jag en JavaScript-fil som bland annat innehöll kod för en hamburgermeny på mobila enheter.

  4. Användning av Git och versionhantering

    För att hålla koll på min kod och mina ändringar använde jag Git. Jag skapade först en huvudgren (main) och började arbeta på mina ändringar där. När jag kom på att jag skulle arbeta i en separat utvecklingsgren för att hålla mina ändringar separerade från produktionen skapade jag en ny gren kallad development. Detta möjliggjorde för mig att testa nya funktioner utan att påverka den stabila koden på main-grenen. Jag använde git checkout -b development för att skapa och byta till min utvecklingsgren.

  5. Automatisering med Parcel och optimering av bilder

    För att optimera webbplatsens prestanda och effektivisera arbetsflödet använde jag Parcel för att automatiskt bygga och optimera mina filer. Jag la också till en bildoptimeringsfunktion som konverterade bilder till moderna format som WebP och AVIF. Jag använde -taggen för att servera olika bildformat beroende på webbläsarens kapabiliteter och säkerställa en bättre användarupplevelse. Jag började med att använda sharp, men insåg att parcel var lättare så därför avinstallerade jag detta program genom att köra npm uninstall sharp.

    Genom detta tillvägagångssätt slipper jag sitta och spara ner varje bild i respektive format för optimering - det räcker med att ha en version av bilden i jpg. Detta sparar enormt mycket tid.

  6. Publicering på Netlify

    För att publicera min webbplats använde jag Netlify, en plattform för att hosta statiska webbplatser. Jag kopplade mitt Git-repo till Netlify och ställde in en byggprocess där Parcel användes för att bygga och optimera alla filer. Vid varje push till min main-gren så byggdes webbplatsen automatiskt och publicerades på Netlify.

  7. Framtida förbättringar

    Jag planerar att lägga till fler funktioner och förbättringar till webbplatsen, bland annat genom att fortsätta använda automatisering för att optimera både prestanda och arbetsflöde. Jag kommer även att fortsätta jobba med att förbättra användarupplevelsen och säkerheten på webbplatsen.
